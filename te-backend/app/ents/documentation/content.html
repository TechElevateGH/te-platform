Platform Overview

TechElevate is a comprehensive talent enablement platform designed to streamline the job search and placement process. The platform serves as a centralized operating system connecting job seekers (members), company employees who can provide referrals (referrers), and TechElevate staff (leads and admins).

The platform facilitates the entire job hunting lifecycle - from resume preparation and application tracking to internal referral requests and skill development through curated learning content.

Mission & Vision

Mission

Democratize access to tech opportunities by providing members with the tools, resources, and network needed to secure their ideal roles in the technology industry.

Vision

Become the leading platform for tech talent enablement, creating a transparent and efficient marketplace connecting aspiring professionals with opportunities.

Core Values

  • Transparency in the job search process
  • Community-driven support
  • Data-driven decision making
  • Continuous learning and growth

User Personas

Member (Job Seeker)

The primary user of the platform. Members are tech professionals or aspiring professionals looking for job opportunities.

Key Capabilities Members can upload and manage multiple versions of their resume, track job applications, request referrals from company insiders, access learning content, and receive feedback from the TechElevate team.

Referrer

Employees at tech companies who are willing to provide internal referrals to qualified members.

  • Review incoming referral requests
  • Evaluate candidate profiles and resumes
  • Provide or decline referrals
  • Track referral status and outcomes

Lead

TechElevate staff members who manage the platform's content and operations.

  • Curate and maintain company database
  • Review member applications and resumes
  • Manage learning content
  • Oversee referral workflows

Admin

Platform administrators with full access to all system capabilities.

  • User management and role assignment
  • System configuration
  • Analytics and reporting
  • Content moderation

Database Design

TechElevate uses MongoDB Atlas as its primary data store, leveraging its flexible document model and cloud-native features.

Data Relationships

MongoDB uses embedded documents and references:

  • Embedded: User profiles embed contact info and preferences
  • Referenced: Applications reference user_id and company_id
  • Denormalized: Company names cached in applications for performance

Authentication & Authorization

The platform uses JWT (JSON Web Tokens) for stateless authentication with role-based access control.

Authentication Flow

  1. User submits email/password credentials
  2. Backend validates credentials against MongoDB
  3. If valid, JWT token is generated with user ID and role
  4. Frontend stores token in localStorage
  5. Token included in Authorization header for subsequent requests
  6. Backend validates token and extracts user information

Referral System

Connect members with employees at target companies to request internal referrals.

How It Works

For Members Browse companies where TechElevate has referrer connections. Submit a referral request with your resume and a personalized note. Track the status of your request and receive notifications when reviewed.
For Referrers Review incoming referral requests from qualified members. Download resumes, evaluate fit, and approve or decline with feedback. Track outcomes of referrals provided.

Referral Request Flow

  1. Member discovers company with available referrer
  2. Member submits request with resume and message
  3. Referrer receives notification and reviews request
  4. Referrer makes decision (approve/decline)
  5. Member receives notification of decision
  6. If approved, referrer submits internal referral
  7. Member applies to job posting

Referral States

  • pending - Awaiting referrer review
  • approved - Referrer will provide referral
  • declined - Referrer declined with reason
  • submitted - Referral submitted to company
  • hired - Member was hired

API Overview

The TechElevate API is a RESTful API built with FastAPI. All endpoints return JSON responses and follow standard HTTP conventions.

Base URL

Production: https://api.techelevate.org/v1
Development: http://localhost:8000/v1

Authentication

Most endpoints require authentication via JWT token in the Authorization header: 

Authorization: Bearer <your_jwt_token>

Response Format

All responses follow a consistent structure:

// Success response
{
  "data": { ... },
  "message": "Operation successful"
}

// Error response
{
  "detail": "Error message",
  "status_code": 400
}

Project Structure Best Practices

  • Keep routers thin: Business logic belongs in CRUD layer or services
  • Use Pydantic models: For request/response validation
  • Dependency injection: For database sessions and auth
  • Error handling: Use custom exceptions from utilities/errors.py
  • Type hints: Always use type annotations

Adding a New Endpoint

# 1. Define schema in ents/[module]/schema.py
class NewItemCreate(BaseModel):
    name: str
    description: str

# 2. Add CRUD operation in ents/[module]/crud.py
async def create_item(db, item: NewItemCreate):
    result = await db.items.insert_one(item.dict())
    return result.inserted_id

# 3. Create endpoint in ents/[module]/endpoints.py
@router.post("/items")
async def create_new_item(
    item: NewItemCreate,
    db = Depends(get_database),
    current_user = Depends(get_current_user)
):
    item_id = await crud.create_item(db, item)
    return {"id": str(item_id), "message": "Item created"}

# 4. Register router in ents/api.py
from .module import endpoints as module_endpoints
api_router.include_router(module_endpoints.router)

Security

Authentication Security

  • Password Hashing: bcrypt with salt rounds
  • JWT Tokens: Short expiration (7 days), secure secret key
  • Token Storage: httpOnly cookies or secure localStorage
  • Session Management: Automatic logout on token expiration

Troubleshooting

Common Issues

Backend Won't Start

Problem ModuleNotFoundError or import errors
  • Verify virtual environment is activated
  • Reinstall dependencies: pip install -r requirements.txt
  • Check Python version: python --version (must be 3.9+)

MongoDB Connection Failed

Problem ServerSelectionTimeoutError
  • Verify MONGODB_URL in .env is correct
  • Check network connectivity to MongoDB Atlas
  • Ensure IP address is whitelisted in Atlas
  • Verify database user credentials

Google Drive Upload Fails

Problem 403 Forbidden or authentication errors
  • Verify GOOGLE_PRIVATE_KEY is properly formatted (with \n)
  • Check service account has domain-wide delegation
  • Ensure GOOGLE_FOLDER_ID exists and is accessible
  • Verify Drive API is enabled in Google Cloud Console

JWT Token Errors

Problem Could not validate credentials
  • Check SECRET_KEY matches between token creation and validation
  • Verify token hasn't expired (check exp claim)
  • Ensure Authorization header format: Bearer <token>
  • Clear browser localStorage and re-login

CORS Errors in Frontend

Problem Access to XMLHttpRequest blocked by CORS policy
  • Verify FRONTEND_URL in backend .env matches frontend origin
  • Check CORS middleware configuration in main.py
  • Ensure backend is running on correct port
  • Check browser console for actual error details

Frontend Build Fails

Problem npm build errors
  • Delete node_modules and package-lock.json
  • Run npm install fresh
  • Check for syntax errors in recent changes
  • Verify Node version compatibility
