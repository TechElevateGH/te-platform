1. Platform Overview

TechElevate is a talent enablement platform designed to help members curate job-hunting assets, request internal referrals, review learning content, and collaborate with the TechElevate operations team. The platform is delivered as a React single-page application backed by a FastAPI microservice and MongoDB datastore.

Mission

Create a centralized operating system for apprenticeships and job placement. Members can manage resumes, applications, and referrals; staff can curate companies, respond to requests, and track pipeline health.

Key Personas

  • Member (Role 1): Manages resumes, applications, and referral requests.
  • Referrer (Role 2): Reviews and fulfills referral requests tied to their company.
  • Volunteer / Lead (Role ‚â•3): Curates company data, triages referrals, oversees learning.
  • Admin (Role 5): Full platform governance; manages users and operations.

Capabilities

  • Resume upload & management with Google Drive storage.
  • Job applications tracking and status workflows.
  • Referral company directory, request submission, and feedback loop.
  • Learning modules, lessons, and practice materials.
  • Admin dashboards for pipeline visibility and analytics.

2. System Architecture

The solution follows a modular three-tier architecture focused on agility, rapid iteration, and compatibility with low-friction cloud tooling.

Frontend (te-frontend)

  • React 18 SPA bootstrapped with Vite.
  • UI kit built with Tailwind utility classes and HeadlessUI primitives.
  • Context providers (AuthContext, DataContext, NotificationContext) manage auth, caching, notifications.
  • Axios instance centralizes API communication and auth header injection.

Backend (te-backend)

  • FastAPI application with modular routers under app/ents.
  • MongoDB Atlas stores users, referrals, applications, and embedded collections.
  • Google Drive used for binary storage of resumes and other documents.
  • Authentication via JWT (AuthJWT) with role-based guards.

Integration & Telemetry

  • Google Drive API for file uploads (application.crud.upload_file).
  • Analytics triggers in frontend (PostHog skeleton) for behavioral tracking.
  • Structured logging middleware for request tracing and error handling.
Request Lifecycle (Example: Resume Upload)
Client submits form ‚Üí Axios sends POST /v1/users/{userId}/resumes with PDF ‚Üí FastAPI validates role, streams file to Google Drive, stores metadata in member_users.resumes array ‚Üí DataContext triggers a refetch ‚Üí UI lists updated resume entry.

4. Primary User Journeys

J1. Member Onboarding

  1. Invite email or self-registration (pending automation).
  2. Complete profile with contact + essay (for referrals).
  3. Upload role-specific resumes via Workspace > Files.
  4. Explore learning modules and track progress.

J2. Requesting a Referral

  1. Browse referral-ready companies (Referrals > Companies).
  2. Confirm prerequisites (resume, essay, contact) flagged in UI.
  3. Submit referral request with job specifics and optional note.
  4. Receive notifications on status changes or feedback.

J3. Referrer Fulfillment

  1. Referrer dashboard surfaces requests for assigned company.
  2. Review candidate materials, provide notes, mark as completed/declined.
  3. System triggers notifications and analytics updates.

J4. Admin Operations

  • Curate company directory and maintain referral requirements.
  • Monitor pipeline via Admin dashboard (files, resumes, essays).
  • Adjust user roles, investigate issues, seed learning content.
  • Export data snapshots for board reporting.

6. Frontend Experience & Modules

Layout & Navigation

  • src/App.jsx wires high-level routes (Workspace, Referrals, Admin).
  • Sidebar & topbar adapt to role-based permissions.
  • Theme + dark mode toggled via DarkModeContext.

State Management

  • AuthContext stores JWT token, user id, role, logout handler.
  • DataContext caches resumes, companies, lessons with refetch flags.
  • NotificationContext polls API for referral + review updates.

Reusable UI Components

  • SlideOvers for creation flows (components/_custom/SlideOver).
  • Form inputs (components/_custom/FormInputs.jsx) unify styling and validation.
  • Toast + SuccessFeedback deliver consistent feedback loops.

Key Screens

  • Workspace: aggregate dashboard, files, resume reviews.
  • Referrals: company browser, my requests, admin oversight.
  • Admin Files: pipeline stats, exports, filtering for staff.
  • Learning: curated lessons, practice sets (scaffolding for expansions).

8. Environment & Configuration

Data Sources

  • MONGODB_URI, MONGODB_DB_NAME

Bootstrap Accounts

  • FIRST_SUPERUSER_EMAIL, FIRST_SUPERUSER_PASSWORD
  • Seeded during app startup if missing.

9. Local Development Workflow

Backend

# Install dependencies
pip install -r requirements.txt

# Run prestart checks (DB connectivity, seed admin)
./prestart.sh

# Start dev server with hot reload
uvicorn app.main:app --reload --port 8000

Frontend

# Install and run
cd te-frontend
npm install
npm run dev  # launches on http://localhost:5173

Use the shared axios configuration in src/axiosConfig.js to align API base URLs between environments. For cross-origin testing, ensure BACKEND_CORS_ORIGINS includes the frontend dev host.

10. Quality & Testing Strategy

  • Unit & Integration Tests: Add to tests/ (backend) and src/__tests__/ (frontend). Use Pytest and Vitest/Jest respectively.
  • API Schema Validation: Expand Pydantic models with stricter field types and field_validator annotations.
  • Manual Verification: Regression pass should cover resume upload, referral submission, notification polling, and admin exports.
  • Static Analysis: Integrate ruff or flake8, plus TypeScript migration for high-safety front-end modules (optional roadmap).

11. Security & Compliance Checklist

  • Rotate service account keys periodically; store secrets in a vault (1Password, GCP Secret Manager).
  • Enforce HTTPS in production via proxy (e.g., Cloudflare, AWS ALB).
  • Restrict MongoDB Atlas IP access to trusted origins.
  • Audit role-based guards in routers (get_current_member_only, require_lead) when adding new endpoints.
  • Enable JWT refresh tokens and session revocation (roadmap item).
  • Document data retention policies for resumes and referral feedback.

13. Glossary & Roles

  • Member: Core user seeking job placement support.
  • Referrer: Internal champion with limited access to referral queue for a single company.
  • Volunteer: Operational staff; can add companies, review content.
  • Lead: Senior staff; broad visibility including analytics and user data.
  • Admin: Full permissions, responsible for platform governance.
  • Resume Review: Separate workflow where members request feedback on documents (handled in resume_reviews router).
  • Workspace: Member dashboard that aggregates files, reviews, and quick actions.
